Getting Started with HTML Notes

Lesson One: Diving In

When learning a new language, where shall we start? Especially when the goal is to communicate with a machine, rather than a human being!!! One intuitive place to begin for newbies is web pages, something that those of us who have only used mices and keyboards are familiar with.

Browser to Server

How do web pages magically appear when we click on a link in our computer's browser? Apparently, when we click on a hyperlink, the browser sends HyperText Transfer Protocol requests via the Internet to the appropriate server. That server then identifies the relevant HyperText Markup Language and reverts via the Internet to our browser, which converts the HTTP into the visual webpage for our viewing pleasure.

HyperText Markup Language

What is this HTML that transfers content between browsers and servers? Apparently, HTML is a language composed of underlying text, references to documents, and links to web pages that browsers and servers understand. The browser reads the underlying HTML text content to create a visual mark-up, which is what we humans see displayed as the web page in our browser.

HTML text content is structured as a tree of HTML elements, each of which are composed of opening tag, content, followed by closing tag. Tags of the same type denote where each element begins and ends.

Browsers Expect Accuracy

When speaking "computer" to a browser, people must be specific as to their HTML syntax. Computers only understand exactly what is conveyed to them, and will not correct for human errors. For example, forgetting to include the closing tag in the HTML text content of an element (excluding void tags) will result in the continuation of that element because the browser has only been told where to start but not where to end.

Grouping Elements: Inline vs. Block

Elements are bifurcated into inline and block elements relating to line breaks as displayed in mark-ups. The former types of elements retains its content in one line. The latter creates an invisible box around the element and cordons it off; thus, when a block element is used, it appears on its own line. Examples of inline elements include "a", "b", "em", "img", "strong", "br" and "span". Examples of block elements include "p", "div" and "form".
